周一

学习的第一讲，先记录下相关的知识点：

设置 GOPATH 的意义在于，可以将其当做是一个或多个工作区，每一个工作区都可以去防止 go 的源码文件以及安装后的归档文件，和可执行文件。
归档文件：在Linux下就是扩展名是.a的文件，也就是archive文件。写过C程序的朋友都知道，这是程序编译后生成的静态库文件。
一般来说，go 语言的源码文件都会被存放到环境变量 GOPATH 包含的某一个目录中 src 目录下

思考题：

Go 语言在多个工作区中查找依赖包的时候是以怎样的顺序进行的？如果在多个工作区中都存在导入路径相同的代码包会产生冲突吗？
会先查询 GOROOT 目录，没有找到依赖包的话会去工作区查找对应的包，且是按照先后顺序查找，查到之后不在查找。查找不到报错。
因此 多个工作区中都存在导入路径相同的代码包 不会冲突

周三

学习的第三讲，先记录下相关的知识点：
go 语言的几个编码规则：代码包声明规则，代码包导入规则，程序实体的访问规则。
go 自己将程序实体的访问权限划分了包级私有和公开，
只有对于函数名称首字母为大写的函数才能被当前包外的代码引用，否则只能被其包内的其他代码引用。
